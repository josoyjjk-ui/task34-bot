# TELEGRAM Cleanup Checklist (운영 고정안)

## 목적
- 텔레그램 대화는 정리하되, 자동화/운영 기억은 보존한다.

## 1) 보존 스냅샷 (삭제 전 필수)
- [ ] USER.md 최신화 확인
- [ ] HEARTBEAT.md 최신화 확인
- [ ] SESSION-STATE.md 최신화 확인
- [ ] MEMORY.md 인덱스/핵심 규칙 확인
- [ ] 크론 상태 확인 (ETF, 아카이빙 작업)
- [ ] 오늘 완료 작업 증빙(run=ok, delivered=true) 확보

## 2) 삭제 분류 기준
### Low Risk (즉시 삭제)
- 잡담/감정성 대화
- 완료된 단건 질의응답
- 중복 진행 보고/중복 알림

### Medium Risk (조건부 삭제)
- 작업 진행 로그
- 외부 링크 포함 대화
- 지시/결과 쌍 중 한쪽만 있는 메시지
- 처리 규칙: 삭제 전 1줄 요약 남기기 (무엇/언제/결과)

### High Risk (보존 후 삭제)
- 고정 템플릿/운영 규칙
- 계정/권한/자동화 지시
- 분쟁 가능성 있는 대화
- 처리 규칙: 로컬 기록 보존 후 삭제

## 3) 실행 순서 (롤링)
1. [ ] 스냅샷 완료
2. [ ] 30일 이전 Low Risk부터 삭제
3. [ ] Medium Risk는 요약본 남긴 뒤 삭제
4. [ ] 삭제 직후 헬스체크
   - [ ] HEARTBEAT 정상
   - [ ] ETF 자동 루틴 정상
   - [ ] 아카이빙 cron 정상
5. [ ] 24시간 관찰 후 다음 구간 삭제

## 4) 실패/이상 대응
- [ ] 자동화 이상 감지 시 즉시 삭제 중단
- [ ] SESSION-STATE/HEARTBEAT 기준으로 복구
- [ ] 복구 완료 전 추가 삭제 금지

## 5) 하트비트 판정 가드 (고정)
- `미완료 경고` 조건: run 증빙 없음
- `HEARTBEAT_OK` 조건: 해당 작업 run=ok + delivered=true 확인됨
