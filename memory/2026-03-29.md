### SmartEditor 이미지 삽입 작업 (블로커)

#### 시도한 방법 (실패)
1. `browser upload tool` → SmartEditor 미연동
2. CDP `DOM.setFileInputFiles` + change 이벤트 → 이벤트 미작동
3. `insertImagesByFile._method.call(inst, [file])` → `getStoreById` 에러
4. `imageBehavior.insert(validated)` → `s is not a function` 에러
5. `uploadImages([{id, source}])` → `[{}]` 반환 (적용 안 됨)
6. DataTransfer drop/paste 이벤트 → 효과 없음
7. `setDocumentData`로 image 컴포넌트 직접 삽입 → "존재하지 않는 이미지" (src 구조 문제)
8. `_uploadImageByBASE64(dataUrl)` → **업로드 성공** (200 OK, 75259 bytes)
9. `_editingService.eraseComponent` → 삭제 성공
10. `insertComponentsWithData([{ctype:'image', src:fullUrl}])` → 삽입은 됐으나 "전송중..." 상태 지속

#### 핵심 발견
- `_uploadImageByBASE64` 성공: 새 이미지 URL 확보
  - `https://blogfiles.pstatic.net/MjAyNjAzMjlfMjUy/MDAxNzc0NzgzMTUyNzYz.7rI--sbr8GcWgQIifmysHVWI3TB8zjGo1TewmqT6rvMg.EOXm5eIXhNOJfNS5MaJkBbpFp4kGeXkODj1kv8xV1V0g.PNG/image.png`
- SE 이미지 컴포넌트 구조 확인:
  - `src`에 path만 넣으면 `http:///path` 오류 → full URL 필요
  - `comp.src = fullUrl` 후 발행하면 깨진 이미지로 저장됨
- `_makeImageComponent`가 native code → 직접 호출 불가
- `insertImagesByUrl({data:[{url, origin}]})` → execute가 false 반환 (데코레이터 체인 문제)
- `insertImages.execute` 데코레이터: `preventOnDocumentProcessing`, `sentryCatch` 외 추가 조건 존재
- `isDocumentProcessing()` = false인데도 execute 실패
- `uploader.insert({validFiles, invalidFiles}, {makeComponent, onEachUpload})` → `_makeImageComponent`가 native code라 블로킹
