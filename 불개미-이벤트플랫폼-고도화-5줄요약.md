# 불개미 이벤트 플랫폼 고도화 — 핵심 5줄 요약

> 작성일: 2026-04-13 | 기준문서: event-section-plan-v2.md

1. **인증/보안 강화 포인트**: Telegram Login Widget 기반 HMAC-SHA256 서명검증으로 실사용자 인증을 도입하고, JWT Access(15분)+Refresh(7일) 토큰 회전·재사용 탐지·family 일괄 폐기 정책과 함께 Turnstile·WAF rate limiting·CSP/HSTS로 다층 보안 체계를 구축한다.

2. **DB 설계 핵심**: users/sessions/consents/events/participants/verification_logs/audit_logs/deletion_requests 8개 테이블 ERD를 설계하여 멱등키(idempotency_key) 기반 중복 참여 방지, 상태 전이 이력 추적(participant_events), soft delete 및 optimistic locking(version)을 통해 개인정보 수명주기와 데이터 무결성을 동시에 확보한다.

3. **Galxe 스타일 UX 차별점**: 게시판 연계 미션(글/댓글/피드백 참여형 퀘스트), 콘텐츠 품질 점수(신고율·추천·운영자 리뷰 기반), 한국어 큐레이션(영어 원문 이벤트 한국어 가이드), 지난 이벤트 학습 아카이브 등 Galxe식 퀘스트보드 UX를 한국 크립토 커뮤니티 맞춤으로 제공한다.

4. **운영 자동화 요소**: 5단계 RBAC(Owner/Admin/Editor/Reviewer/Member)와 draft→review→publish 승인 워크플로우, CSV dry-run import, 예약 게시 배치, 이벤트 버전 rollback, 무료 한계 70/85/95% 임계치 자동 알람, D1 p95 지연·5xx율 기반 캐시 TTL·aggregate 분리 자동 스케일링 정책을 적용한다.

5. **Supabase 전환 이점**: 현재 D1 기반 아키텍처에 DB 추상화 계층을 두어, 트래픽 증가 시 Supabase(PostgreSQL)로 전환하면 실시간 구독(Realtime), Row Level Security 기반 세밀 권한 제어, 내장 Auth·Storage·Edge Functions 활용으로 운영 복잡도를 줄이고 DAU 5만 피크 시나리오까지 안정적으로 확장할 수 있다.
